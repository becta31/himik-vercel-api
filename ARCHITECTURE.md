# Архитектура проекта «ХИМИК»

## 1. Назначение проекта

**ХИМИК** — это интерактивный образовательный веб-тренажёр по химии.

Проект помогает пользователю:

- изучать химические элементы;
- искать элементы по названию или символу;
- просматривать подробную информацию об элементах;
- проходить викторину;
- сохранять результат прохождения;
- смотреть таблицу лидеров.

Архитектура проекта построена как простое serverless-приложение: статический frontend размещается на Vercel, а динамическая часть рейтинга работает через Vercel Serverless Functions и MongoDB Atlas.

---

## 2. Общая схема

```text
Пользователь
   ↓
Браузер
   ↓
HTML / CSS / JavaScript
   ↓
Vercel Static Hosting
   ↓
Vercel Serverless Functions
   ↓
MongoDB Atlas
```

## 4. Frontend

Frontend реализован без фреймворков, на обычных HTML-страницах и Vanilla JavaScript.

Используемые технологии:

- HTML5;
- CSS;
- Tailwind CSS CDN;
- Vanilla JavaScript;
- Google Fonts;
- Vanta.js на главной странице.

### 4.1 Главная страница — `index.html`

Главная страница выполняет роль входной точки.

Она содержит:

- название проекта;
- краткое описание;
- переход в режим обучения;
- переход к викторине;
- навигацию по сайту;
- визуальный фон с анимацией.

### 4.2 Режим обучения — `study.html`

Страница обучения загружает данные из файла:

```text
elements.json
```

Основная логика:

- загрузка массива химических элементов;
- отрисовка карточек элементов;
- поиск по русскому названию, английскому названию и символу;
- фильтрация по категориям;
- отображение подробной информации в модальном окне.

Данные обрабатываются полностью на стороне браузера.

### 4.3 Викторина — `quiz.html`

Страница викторины также загружает элементы из:

```text
elements.json
```

Основная логика:

- генерация случайных вопросов;
- выбор типа вопроса;
- формирование вариантов ответа;
- проверка выбранного ответа;
- подсчёт очков;
- подсчёт правильных ответов;
- подсчёт максимальной серии правильных ответов;
- отправка результата в API после завершения;
- загрузка и отображение таблицы лидеров.

## 5. Статические данные

Файл:

```text
elements.json
```

хранит список химических элементов.

Каждый элемент содержит поля:

```json
{
  "atomic_number": 1,
  "symbol": "H",
  "name_ru": "Водород",
  "name_en": "Hydrogen",
  "mass": 1,
  "category": "nonmetal",
  "period": 1,
  "group": 1,
  "electron_configuration": "1s¹",
  "description": "Самый распространенный элемент во Вселенной."
}
```

Эти данные используются:

- на странице обучения;
- в викторине;
- для генерации вопросов;
- для отображения карточек элементов.

## 6. Backend API

Backend реализован через **Vercel Serverless Functions**.

Каждый файл в папке `/api` становится отдельным HTTP-эндпоинтом.

### 6.1 Сохранение результата

Файл:

```text
api/submit-result.js
```

Эндпоинт:

```text
POST /api/submit-result
```

Назначение:

- принять результат викторины;
- проверить обязательные поля;
- ограничить имя пользователя до 50 символов;
- подключиться к MongoDB;
- сохранить результат в коллекцию `QuizResults`;
- вернуть JSON-ответ.

Пример тела запроса:

```json
{
  "userName": "Денис",
  "score": 80,
  "correctAnswers": 8,
  "maxStreak": 4
}
```

Пример успешного ответа:

```json
{
  "success": true,
  "insertedId": "..."
}
```

### 6.2 Получение рейтинга

Файл:

```text
api/get-top-results.js
```

Эндпоинт:

```text
GET /api/get-top-results
```

Назначение:

- подключиться к MongoDB;
- получить записи из коллекции `QuizResults`;
- отсортировать результаты по убыванию очков;
- при одинаковых очках учитывать более свежие результаты;
- вернуть топ-10 результатов.

Сортировка:

```js
.sort({ score: -1, timestamp: -1 })
```

Ограничение:

```js
.limit(10)
```

## 7. База данных

В проекте используется **MongoDB Atlas**.

Текущая база данных в коде:

```text
Cluster0
```


Документ результата имеет структуру:

```json
{
  "_id": "ObjectId",
  "userName": "Денис",
  "score": 80,
  "correctAnswers": 8,
  "maxStreak": 4,
  "timestamp": "Date"
}
```

---

## 8. Переменные окружения

Для подключения к MongoDB используется переменная окружения:

```text
MONGO_URI
```

Она должна быть настроена в Vercel:

```text
Project Settings → Environment Variables
```

Пример формата:

```text
mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority&appName=Cluster0
```

Важно:

- `MONGO_URI` не хранится в GitHub;
- строка подключения не должна попадать в публичный код;
- после изменения переменной окружения нужно выполнить redeploy проекта.

## 9. Поток данных: режим обучения

```text
Пользователь открывает study.html
   ↓

   ↓
JavaScript сохраняет элементы в массив
```

## 10. Поток данных: прохождение викторины

```text

   ↓
Браузер загружает elements.json
   ↓
JavaScript генерирует вопросы
   ↓
Пользователь отвечает на 10 вопросов
   ↓
JavaScript считает score, correctAnswers и maxStreak
   ↓

   ↓
POST /api/submit-result
   ↓
Serverless Function сохраняет результат в MongoDB
   ↓
API возвращает success: true
```

## 12. Безопасность

В проекте реализованы базовые меры безопасности:

### 12.1 Защита секретов

Строка подключения к MongoDB хранится только в переменной окружения:

```text
MONGO_URI
```

Файл `.gitignore` исключает:

```text
.env
.env.local
.env.*.local
.vercel/
node_modules/
```

### 12.2 Проверка данных на backend

`submit-result.js` проверяет наличие обязательных полей:

- `score`;
- `correctAnswers`;
- `maxStreak`.

Если данных нет, API возвращает ошибку `400`.

### 12.3 Ограничение имени пользователя

Перед сохранением имя пользователя приводится к строке, обрезается и ограничивается 50 символами.

### 12.4 Экранирование имени в рейтинге

Перед выводом имени пользователя в таблицу лидеров frontend экранирует HTML-символы через функцию:

```js
escapeHTML()
```

Это снижает риск HTML-инъекции при отображении рейтинга.

### 12.5 CORS

API добавляет CORS-заголовки, чтобы frontend мог обращаться к serverless-функциям.

## 13. Обработка ошибок

### Frontend

На frontend предусмотрены:

- fallback-данные для викторины, если `elements.json` не загрузился;
- сообщение об ошибке при невозможности загрузить рейтинг;
- логирование результата отправки викторины в консоль.

### Backend

Serverless API возвращает JSON-ответы:

При успешной записи:

```json
{
  "success": true,
  "insertedId": "..."
}
```

При ошибке валидации:

```json
{
  "success": false,
  "message": "Missing required fields in request body."
}
```

При ошибке сервера:

```json
{
  "success": false,
  "message": "Internal Server Error",
  "error": "..."
}
```

## 14. Ограничения текущей версии

Текущая версия проекта не использует:

- авторизацию пользователей;
- личные кабинеты;
- роли пользователей;
- админ-панель;
- защиту от повторной отправки результатов;
- серверную проверку корректности результата викторины;
- полноценную сборку frontend;
- backend-фреймворк.

Также frontend подключает Tailwind CSS через CDN. Для production-версии лучше перейти на сборку Tailwind через build step.

## 16. Итог

Проект «ХИМИК» реализован как простое и понятное serverless-приложение:

- frontend работает как статический сайт;
- данные элементов хранятся в `elements.json`;
- результаты викторины сохраняются через Vercel API;
- рейтинг хранится в MongoDB Atlas;
- таблица лидеров обновляется после прохождения викторины.
