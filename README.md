## 🚀 README: HIMIK - Тренажер по химии

Тренажер по химии, разработанный с использованием **HTML, Tailwind CSS и чистого JavaScript**.

### 📁 Структура проекта

```
HIMIK/
├── index.html        # Главная страница, меню
├── study.html        # Режим обучения (периодическая таблица)
├── quiz.html         # Режим викторины
├── elements.json     # Данные о химических элементах (JSON)
└── README.md         # Этот файл
```

-----

## 🛠️ Как запустить проект

### Способ 1: Прямое открытие в браузере (Самый простой)

1.  **Найдите** файл `index.html` в папке проекта.
2.  **Дважды кликните** по файлу `index.html`, чтобы открыть его в браузере.

### Способ 2: Запуск локального веб-сервера (Для обхода блокировки JSON)

Если у вас возникли проблемы с загрузкой данных из `elements.json` (ошибка CORS), используйте простой HTTP-сервер.

1.  **Откройте** командную строку (или терминал) в корневой папке проекта.
2.  **Запустите** сервер с помощью Python (если установлен):
    ```bash
    python -m http.server
    ```
3.  **Откройте** браузер и перейдите по адресу:
    ```
    http://localhost:8000/
    ```

## 🌳 Контроль версий с помощью Git

Для отслеживания изменений, совместной работы и резервного копирования кода используется система контроля версий **Git**.

### 1\. Клонирование репозитория (Первый запуск)

Если проект уже находится на GitHub, используйте эту команду, чтобы получить локальную копию:

1.  **Откройте** терминал.
2.  **Перейдите** в папку, куда хотите сохранить проект.
3.  **Выполните** команду (замените `[URL_РЕПОЗИТОРИЯ]` на актуальный адрес):
    ```bash
    git clone [URL_РЕПОЗИТОРИЯ]
    ```
4.  **Перейдите** в папку проекта:
    ```bash
    cd HIMIK
    ```

### 2\. Рабочий процесс Git (Сохранение изменений)

После того как вы внесли изменения в файлы (`study.html`, `elements.json` и т.д.):

| Шаг | Команда | Назначение |
| :--- | :--- | :--- |
| **Проверить статус** | `git status` | Показывает, какие файлы были изменены. |
| **Добавить изменения** | `git add .` | Добавляет все измененные файлы в область подготовки (staging area). |
| **Зафиксировать изменения** | `git commit -m "Краткое описание изменений"` | Создает точку сохранения (коммит) в вашей локальной истории. |
| **Отправить на GitHub** | `git push origin [имя_ветки]` | Отправляет все локальные коммиты в удаленный репозиторий (например, на GitHub). Для основной ветки обычно используется `main` или `master`. |

### 3\. Получение последних обновлений

Если над проектом работает несколько человек, или вы работали на другом компьютере, используйте эту команду для загрузки последних изменений:

```bash
git pull origin [имя_ветки]
```
