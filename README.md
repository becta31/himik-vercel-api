## 🏗 Архитектура

Проект построен на архитектуре **Serverless Jamstack**:
*   **Frontend:** Статические файлы (HTML, CSS, JS), размещенные на CDN Vercel.
*   **Backend:** Serverless функции (Node.js), работающие по запросу.
*   **Database:** MongoDB Atlas для хранения результатов.

## 🚀 Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/becta31/himik-vercel-api.git
cd himik-vercel-api
```

### 2. Локальный запуск
Так как проект использует `fetch` для загрузки данных (`elements.json`), его нужно запускать через локальный веб-сервер, а не просто открывать файлы в браузере.

**Вариант А: Использование VS Code (Live Server)**
1. Установите расширение **Live Server** в VS Code.
2. Откройте корневую папку проекта.
3. Нажмите кнопку **"Go Live"** в нижней строке состояния.

**Вариант Б: Использование Python (если установлен)**
```bash
# Python 3
python -m http.server 8000
```
Затем откройте `http://localhost:8000` в браузере.

### 3. Настройка переменных окружения (для Backend)
Для работы API локально или при деплое необходимо указать переменные окружения в Vercel:
*   `MONGO_URI`: Строка подключения к MongoDB Atlas.
*   `MONGO_DB_NAME`: Имя базы данных (опционально, по умолчанию `Cluster0`).

## 📖 Документация API

### Сохранить результат
*   **URL:** `/api/submit-result`
*   **Method:** `POST`
*   **Body:**
    ```json
    {
      "userName": "Игрок",
      "score": 150,
      "correctAnswers": 8,
      "maxStreak": 3
    }
    ```

### Получить топ-результаты
*   **URL:** `/api/get-top-results`
*   **Method:** `GET`
*   **Response:** Массив объектов топ-10 игроков.
